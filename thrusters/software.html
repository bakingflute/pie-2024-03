Software System

The software system for Ghost Thrusters is responsible for controlling the balance and stability of the system using a PID-based approach. It incorporates multiple thrusters, each programmed to respond dynamically to changes in orientation and velocity. The system's architecture ensures precision adjustments to maintain desired angles and mitigate external disturbances.

Key Features

  • PID-based thrust control for maintaining target pitch and roll.
  • Dynamic counter-thrust to correct overcompensation.
  • Modular code design for extensibility and debugging.

How It Works

The control system operates by calculating the error between the desired orientation (setpoint) and the current orientation of the system. Using a PID (Proportional-Integral-Derivative) controller, it computes the necessary thrust adjustments for each thruster to minimize this error. The incorporation of velocity feedback ensures that the system accounts for dynamic changes, reducing overshoot and avoiding oscillations. By adapting thrust based on real-time velocity and position, the system achieves smoother and more precise corrections.

Earlier approaches relied solely on angular error without factoring in velocity, leading to frequent overcompensation and instability. This resulted in jerky movements and slower stabilization times. The current design leverages velocity to predict system behavior and apply counter-thrust only when necessary, achieving a more fluid and efficient balance. Additionally, the use of dynamic thresholds and damping factors further reduces unnecessary thruster activity, enhancing the overall performance and responsiveness of the system.

Challenges

Implementing a real-time control system presented challenges such as tuning PID parameters for optimal response and ensuring precise synchronization of multiple thrusters. Additionally, managing counter-thrust logic to prevent oscillations required extensive testing and debugging.

Future Improvements

  • Enhance counter-thrust logic for faster stabilization.
  • Implement auto adjustment logic for adaptive PID parameter tuning.
