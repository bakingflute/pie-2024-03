Firmware Systems Description

In order to power the electrical systems we have and control them remotely, we are using two Arduinos. Only one of them, which has an Adafruit Motor Shield on it, is used for firmware purposes. This Arduino with the motor shield is responsible for powering the drivetrain, providing the ability to swap between 33.3 and 45 rotations per minute (rpm), and communicating the current rpm to a Matrix Portal M4, which controls the Adafruit RGB display on the inside of our box's lid.
The rpm control is provided by a physical switch which interfaces with the Arduino and is located on the lower inside panel of the box. Next to it is another physical switch that can interrupt the power going from the Arduino to the drivetrain. The Arduino communicates information to the Matrix Portal M4 via serial connection, which results from wiring the RX and TX pins to each other on both devices and wiring their grounds together as well.
Near the switches described above, there is a volume knob with a potentiometer underneath that connects into the sound circuit taking cartridge output to the speaker. By turning the knob and varying the resistance of that circuit component, the user can control the speaker's volume. The sound circuit is powered via an Arduino that does not have any code loaded on it.
