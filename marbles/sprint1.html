SPRINT 1 GOAL

In sprint one, we wanted to see if we could sense and sort the marble into two columns in an unintegrated system.

In this sprint we wanted to see if we could model and protoype most parts of our system in an isolated way to see if they could work on their own. We also wanted to try some simple integration into a very rudimentary version of our system with only one sorter and two paths

MECHANICAL

The mechanical subteam's sprint one goal was to prototype a marble pathway, rudimentary sorting mechanism, and rudimentary marble frame.

All mechanical goals were achieved during this sprint. The first marble pathway was prototyped to allow color sensing and solenoid testing. The pathway is a chamber that allows marbles to fall into a chamber where they are color sensed and bumped over a ledge with a solenoid into a sorting mechanism and eventually frame. This first pathway did not include any front plate but had basic attachments for the color sensor and solenoid so testing could be performed. The pathway mechanism was kept separate from the sorting mechanism and frame. A rudimentary sorting mechanism and frame were also prototyped. The sorting mechanism, operated by a rear-mounted servo, could sort two columns with accuracy and four after minor modification. This sorting mechanism was attached to a frame with two columns and allowed us to test marble sorting. Together, the sorting mechanism and frame were attached to a basic container that could hold the marbles once dropped, and the frame upright.

ELECTRICAL

The electrical subteam’s sprint one goal was to prototype a color sensor and solenoid mechanism. we also wanted to draw up a rudimentary wiring diagram for what we thought the system would need

All electrical goals were achieved during this sprint. The color sensor and solenoid mechanism were prototyped and tested. The color sensor was able to detect the color of the marbles and send a signal to the solenoid to bump the marble into the sorting mechanism. The color sensor was mounted to the pathway and the solenoid was mounted to the pathway as well. The solenoid was able to bump the marble into the sorting mechanism with accuracy. The color sensor was able to detect the color of the marbles with accuracy. A rudimentary wiring diagram was also drawn up. The wiring diagram was a basic layout of what we thought the system would need to function. The diagram included the color sensor, solenoid, and servo. The diagram was used to help us understand what we needed to do to get the system to work.

SOFTWARE

The software goal was to ensure each electrical component for color sensing and sorting could individually function as we intended. We broke this down into three steps: actuate the solenoid to push a marble off a ledge on command, determine the color of a marble using a color sensor, and move a servo motor to feed the marble into a different column based on the determined color of the marble.

We mostly reached this software goal. We wrote two programs in the Arduino IDE: one to determine marble color using the color sensor, and another to actuate the servo motor and solenoid as if the color of the marble was known. The actuating program confirmed that the solenoid and servo motor would function as we intended, but the color sensing program made errors in determining the marble’s color – more than we believed would work for our project. The program to determine marble color was calibrated by analyzing the sensor readings of each marble color, and based on this, manually making rules to categorize future sensor readings into a known marble color. By testing the calibration, we found that the marble’s color was often incorrectly categorized. The marble color would need to be nearly perfectly categorized for the marble machine to make art, and because this did not occur, we did not fully reach our goal for color sensing.
